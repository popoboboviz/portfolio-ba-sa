# Требования — Страница брендов одежды

## Цель
Сделать страницу брендов категории «Одежда», где пользователь может:
- посмотреть список брендов (логотип + название),
- отсортировать бренды по алфавиту,
- найти бренд по названию,
- выбрать бренд и перейти на страницу товаров/баннеров бренда.

## Участники
- **Пользователь** — просматривает/ищет/сортирует/выбирает бренд.
- **Система** (Front + сервисы) — получает данные и отображает пользователю.

## Предусловия
- Пользователь открыл страницу брендов одежды.
- В системе есть данные о брендах (название, логотип, привязка к коллекции товаров).

## Данные бренда
- **name** — уникальное название бренда
- **logoUrl** — ссылка на логотип
- **collectionId** — идентификатор коллекции товаров, связанной с брендом

---

## Функциональные требования (FR)

### FR-1 Просмотр брендов списком
**Описание:** система показывает список брендов (логотип + название).

**Основной сценарий:**
1) Пользователь открывает страницу брендов.
2) Система запрашивает список брендов.
3) Система отображает список.

**Ошибки:**
- Если список не загрузился (сервис недоступен) — показать сообщение «Не удалось загрузить бренды».

---

### FR-2 Сортировка брендов A–Z
**Описание:** пользователь может отсортировать бренды в алфавитном порядке.

**Сценарий:**
1) Пользователь нажимает «Сортировка A–Z».
2) Система получает/перестраивает список с сортировкой по имени.
3) Система отображает отсортированный список.

---

### FR-3 Поиск бренда по названию
**Описание:** пользователь вводит строку поиска и получает бренды, подходящие по названию.

**Сценарий:**
1) Пользователь вводит название бренда в поле поиска.
2) Система выполняет поиск по названию.
3) Система показывает результат.

**Ветвления:**
- Если ничего не найдено — показать «Ничего не найдено».
- Если найдено — показать список найденных брендов.

---

### FR-4 Выбор бренда и переход в товары бренда
**Описание:** при выборе бренда система показывает товары и баннеры бренда.

**Сценарий:**
1) Пользователь выбирает бренд.
2) Система запрашивает товары и баннеры по `collectionId`.
3) Система отображает страницу бренда: товары + баннеры.

**Ветвления:**
- Если товары не найдены — показать пустую страницу бренда / сообщение «Товаров нет».

---

## Нефункциональные требования (NFR) — минимально
- UI должен отображать логотип и название для каждого бренда.
- Ошибки сервиса должны отображаться понятным сообщением пользователю.
